# 第一性原则

## 目标函数

把“可读的小说文本”转为“可拍、可审、可连载、可分镜”的工程化剧本资产。

## 四个不可妥协约束

1. 可拍性：所有关键信息必须可视化（动作/镜头/台词/音效）
2. 留存性：单集必须具备钩子、冲突、反转、悬念
3. 一致性：角色、人设、道具、关系、时间线不可断裂
4. 可控性：所有阶段都有可验证门控（文件存在+规则检查+PASS/FAIL）

## 两个核心创作原则

### 1. 聚焦主线原则

**核心主旨**：所有情节、角色、对话必须推进主线剧情

**执行标准**：
- 不为剧情前进服务的无用情节必须删掉
- 不为剧情前进服务的无用角色必须删掉
- 不为剧情前进服务的无用对话必须删掉
- 任何让节奏变慢的部分，都要毫不犹豫地砍掉

**判定方法**：
- 问自己：这个情节/角色/对话是否推进主线剧情？
- 如果答案是"否"，立即删除
- 如果答案是"不确定"，也删除

### 2. 一句一拳原则

**核心主旨**：**一句一拳，每一句都在做事**

**执行标准**：
- 每一句台词都必须推进剧情、展现人物或制造冲突
- 没有废话，没有闲聊，没有无意义的寒暄
- 每一句话都要有信息增量

**判定方法**：
- 问自己：这句台词做了什么事？
  - 推进剧情？
  - 展现人物性格？
  - 制造冲突？
  - 揭示信息？
- 如果答案是"什么都没做"，立即删除

**反例**：
```
❌ 无用对话：
A："你好。"
B："你好。"
A："今天天气不错。"
B："是啊。"
```

**正例**：
```
✅ 有用对话：
A："萧家的人来了。"（推进剧情）
B："我不怕他们。"（展现人物）
A："你会后悔的。"（制造冲突）
```

## 统一流程抽象

1. 输入：原文/剧情
2. 结构化：分析与建模
3. 规划化：分集与情绪曲线
4. 生成化：单集剧本
5. 门控化：业务+合规审核
6. 桥接化：分镜/动态提示词
